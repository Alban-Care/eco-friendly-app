Conception et technologies

Dictionnaire de données décrit les données à stocker dans la base de données.

Il permet de définir les entités et leurs attributs, en détaillant leur :

  • type
  • taille
  • contraintes
  • etc

Conception et technologies

La modélisation des données

Dictionnaire de données : Les entités

Exemple : Entité User

7 entités :

  • User
  • Category
  • Article
  • Advice
  • Quiz
  • Answer
  • Avatar

MCD décrit les données sous forme de schémas entité-association

Les entités représentent les objets réels stockés dans la base de données, et les associations décrivent les relations entre ces objets par un verbe et avec les cardinalités.

Conception et technologies

La modélisation des données

Modèle conceptuel de données (MCD)

MLD permet de traduire le MCD en un schéma relationnel

Les entités deviennent des tables, tous les champs sont listés et les associations deviennent des clés étrangères reliant les tables.

Conception et technologies

La modélisation des données

Modèle logique de données (MLD)

MLD permet de préciser les détails techniques de la base de données, tels que les types de données, les contraintes d'intégrité et les index.

Conception et technologies

La modélisation des données

Modèle physique de données (MPD)

L'architecture back-end suit le modèle MVC (Modèle-Vue-Controleur)

Le modèle MVC est un modèle d'architecture logicielle qui sépare les données de l'application, la logique métier et l'interface utilisateur.

Il permet de :

  • séparer les différentes responsabilités de l'application
  • de faciliter la maintenance et l'évolution de l'application
  • de faciliter le travail en équipe.

Le modèle représente les données, la vue la présentation des données et le contrôleur gère les actions utilisateurs.
